# 3.2：最后的加固与项目收官

**目标**：完善页面访问权限，清理代码中的“蛛丝马迹”，并完成项目的正式打包，养成职业开发者习惯。

---

### 第一步：权限守卫——不登录不准看

**讲解**：目前我们的“页面跳转”全靠手动点。如果黑客直接在代码里改 `currentPage.value = 'list'`，他还是能绕过登录。我们要写一个“保安”逻辑。

1. 在 `<script setup>` 中，修改我们之前的 `savedUser` 读取逻辑，增加一个强制检查：
    
    ```jsx
    // 页面启动时的“保安”检查
    const checkAuth = () => {
      const savedUser = localStorage.getItem('my_app_user')
      if (!savedUser && currentPage.value !== 'login') {
        // 如果没登录 且 不在登录页，强制回登录页
        currentPage.value = 'login'
        alert('请先登录！')
      }
    }
    
    // 执行检查
    checkAuth()
    ```
    

---

### 第二步：逻辑加固——防止“偷梁换柱”（CSRF 意识）

**讲解**：假设你在别的网页点了一个中奖链接，那个链接可能会悄悄控制你的浏览器，在你的搭子广场发一条垃圾广告。这就是“跨站请求伪造”。

1. **防御小技巧：二次确认**：
对于发布、删除等关键操作，多弹一个确认框，虽然麻烦一点点，但能防止脚本自动误操作。
2. **实习操作**：检查你的 `addBuddy`（发布）和 `deleteBuddy`（删除）函数，确保都有 `confirm()` 确认步骤。
    
    ```jsx
    const addBuddy = () => {
      if (!confirm('确定发布这条邀请吗？')) return
      // ... 原有的发布逻辑
    }
    ```
    

---

### 第三步：清理“蛛丝马迹”（代码审计）

**讲解**：在正式把网页给别人看之前，我们要把测试时的痕迹删掉，否则黑客能通过控制台看到你的逻辑。

1. **查找并删除 `console.log`**：
    - 在 VS Code 中按 `Ctrl + Shift + F`（全局搜索）。
    - 输入 `console.log`。
    - **动作**：删掉所有打印账号、密码、Token 的语句。
2. **检查环境变量**：
    - 确保你的 `.env` 文件没有被误传到网上（虽然实习环境没关系，但要有这个意识）。

---

### 第四步：打包上线——看看真正的网页长啥样

**讲解**：我们平时运行的是“开发版”，代码是乱糟糟的。现在我们要把它压缩成体积最小、速度最快的“生产版”。

1. 在终端（Terminal）中输入并回车：
    
    ```bash
    npm run build
    ```
    
2. **观察变化**：
    - 你的项目文件夹里多了一个 `dist` 文件夹。
    - 打开 `dist/assets` 里面的 `.js` 文件。
    - **现象**：你会发现代码变得密密麻麻，变量名都变成了 `a`, `b`, `c`。
    - **安全意义**：这种“混淆”让黑客很难一眼看懂你的业务逻辑，增加了攻击难度。

### 第六步：结课寄语

这三天，各位亲手做出了一个有登录、有发布、有逻辑校验的校园搭子平台。
你们学到的不仅是 Vue 的 `ref`、`v-for`、`v-model`，更重要的是建立了一种“开发者思维”：

1. **数据驱动**：网页是数据的影子。
2. **防御性编程**：假设用户会乱填数据，假设黑客会尝试攻击。
3. **极简主义**：能用 `{{ }}` 就不用 `v-html`。
